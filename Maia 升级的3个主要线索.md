我想要新增一个 Agent，作为对用户的想法的评估者，它是一个隐藏在背后的“法官”，在用户与 interviewer 对话交流的过程之中，在几个关键的节点对用户的想法进行评估和判断，并且给出意见，意见不是直接给到用户（用户不与这个 Agent 直接互动），意见是像 system notice 一样给到 interviewer 的，又 interviewer 知晓并且作为和用户沟通交流的角色，大致来说就是 interviewer 得到了来自评估者 Agent 的评估意见之后，实时地调整他的话语和后续提问的内容和建议的内容，实现对用户的更好的引导和思维的纠偏。
那么问题来了，这个评估者 Agent 要评估的是什么呢？
大概是以下：
第一个目标：
得到“具体且真实的需求/痛点/难题”；
什么叫具体？如何算真实？
不讨论评判方法，而是设计处理手段——得到一系列想法，剥离掉这些想法的“工具形状”，
将原始想法经过清洗和加工，得到一系列对于“具体且真实的需求/痛点/难题”的陈述。
区分“表面需求”（工具构想）和“本质需求”（终极目的——最终想达到什么状态？）。
自查公式：我只是想要___，而不需要关心___。
示例：
我选择“每周重复填写格式僵化的部门周报”作为最值得解决的场景。
场景描述：作为市场专员，每周五下午需花费1小时，从微信聊天记录、Excel数据表、邮件汇报中零散提取信息，手动拼接成固定格式的Word周报。最令人崩溃的是，领导频繁调整汇报模板，每次修改都需重新协调历史数据，极易出错。
本质需求：我需要的不是更快的复制粘贴工具，而是一种能自动聚合多平台工作痕迹、并灵活适配模板变化的“动态汇报助手”——让我能专注于分析数据价值，而非重复整理格式。若能实现，团队每人每周可节省至少2小时机械劳动，将时间投入到策略优化中。
“具体”的黄金法则：
能拍成一部 3 分钟短视频——如果能让他人（尤其是不相关领域的人）像看短片一样在脑中清晰地复现场景，就叫具体。
有明确的主角和行动：用户是谁？他正在做什么？
有时空和流程细节：在什么时间、地点？分几步完成？哪一步是瓶颈？
有可以感知的坏结果：结局是什么？重复了、出错了、耽误了、气疯了？
“真实”的钢铁标尺：
它在造成可以被测量的损失。一个需求/痛点/难题，如果移除所有技术幻想后，它仍在持续造成资源浪费，就叫真实。
损失可被客观描述吗？
时间损失：是否每周/月固定耗费X小时？（例如：“每周合计浪费至少3小时。”）
金钱/机会损失：是否导致直接成本增加或业务机会错失？（例如：“因响应慢而丢失潜在客户。”）
精力/情绪损耗：是否让你感到烦躁、厌倦或注意力被无价值占用？
它是一个“现状”，而非“幻想”吗？
不要：“要是有个AI能帮我做战略决策就好了。”（这是幻想，是“想要一艘飞船”）
要：“我每天要花2小时手动整理散落在聊天记录、邮件和表格里的客户需求，才能开始做分析。”（这是现状，是“正在用铲子挖土搬石头”）
它有“不得不做”的强制性吗？
真实：即使痛苦、低效，但这件事目前为了生存/工作/生活必须完成，没有选择。（如：报销、周报、数据同步）
不真实：这件事可以被完全放弃或忽略，而不产生实质性后果。
